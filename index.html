Diagrama de Relacionamentos (ER)
🔗 Relacionamentos entre Tabelas
usuarios (1) ──── (N) personagens  → Um usuário pode ter vários personagens
personagens (1) ──── (1) progresso_jogador  → Cada personagem tem um progresso de sessão
personagens (1) ──── (N) inventario  → Personagem possui múltiplos itens
salas (1) ──── (1) inimigos  → Cada sala tem um inimigo fixo
progresso_jogador (1) ──── (N) log_combate  → Progresso registra todos os turnos
log_combate (N) ──── (1) salas  → Log referencia a sala do combate
Arquitetura Java — Classes Principais
Classe
☕ Main.java
Ponto de entrada. Inicializa o jogo e gerencia o fluxo principal de cenas.
Classe
🔐 AuthService.java
Gerencia login, validação de credenciais e hash de senha. Conecta com tabela usuarios.
Classe
⚔ Personagem.java
Representa o herói. Atributos: nome, classe, HP, ataque, defesa. Métodos: atacar(), curar().
Classe
👹 Inimigo.java
Representa cada monstro da dungeon. Herda de Entidade. Atributos: nome, HP, ataque.
Classe
🏰 Dungeon.java
Controla a progressão de salas. Instancia inimigos e gerencia transição entre cenas.
Classe
⚡ SistemaCombate.java
Loop de turnos. Processa ações: atacar e usar poção. Calcula dano e registra log.
Classe
📖 Narrador.java
Exibe texto lentamente com efeito de digitação. Aplica cores ANSI ao terminal.
Classe
🗄 DatabaseManager.java
Gerencia conexão JDBC com o banco. Executa queries de CRUD para todas as tabelas.
Classe
🧪 Inventario.java
Gerencia itens do personagem. Método usarPocao() restaura HP durante combate.
Fluxo de Dados — Terminal ↔ Java ↔ Banco
💻 Terminal
Entrada do usuário via Scanner.
Saída colorida via ANSI codes.
☕ Camada Java
Lógica de negócio, combate, narrativa e controle de fluxo.
🗄 Banco de Dados
MySQL / SQLite via JDBC.
Persiste progresso e logs.
🔄 Ciclo de Dados por Turno de Combate
1. Jogador digita opção no terminal (1=Atacar / 2=Poção)
2. SistemaCombate processa a ação e calcula dano
3. Inimigo responde com ataque aleatório dentro do range
4. HPs são atualizados em memória (Personagem e Inimigo)
5. DatabaseManager salva turno em log_combate
6. Narrador exibe resultado colorido no terminal
7. Verificação de condição de vitória/derrota → próxima sala ou Game Over
